# ELKASS PROJECT BOOK

## Wizja

ELKASS Platform ma być jednocześnie:

1. pięknym sklepem RTV/AGD dla klientów,
2. prostym narzędziem dla pracowników,
3. własnym oprogramowaniem ELKASS, które można rozwijać przez lata.

## Główne zasady

- Klient nie widzi technicznych informacji o Builderach.
- Wszystko, co klient widzi, ma wyglądać jak gotowy sklep premium.
- Pracownik ma edytować treści z ELKASS Studio bez kodu.
- Desktop i mobile są rozwijane równolegle.
- Motywy sezonowe mają działać elegancko, bez tandety.
- Produkty, ceny, grafiki i promocje mają pochodzić z jednego źródła danych.

## Ustalenia projektowe

### Home

Home ma być historią marki ELKASS, a nie zbiorem przypadkowych sekcji. Kolejność:

- Header + Hero,
- Hit tygodnia i promocje,
- producenci,
- kategorie,
- Poznaj ELKASS,
- opinie,
- kontakt / mapa.

### Opinie

Opinie mają być kompaktowe, wiarygodne i lokalne. Widoczne jako rotujący moduł z wieloma opiniami, imieniem oraz inicjałem nazwiska.

### Produkt

Karta produktu ma sprzedawać produkt, nie usługi. Dlatego:

- zdjęcie i najważniejsze parametry są wysoko,
- cena i CTA są czytelne,
- ekspert i usługi ELKASS są niżej,
- finansowanie ma dwie osobne opcje: Raty 0% i Dogodne raty.

### Mobile

Mobile nie jest tylko zwężonym desktopem. Każdy moduł ma być oceniany osobno na telefonie.

## Bieżący sprint

Sprint 0 + V44 porządkuje kartę produktu i przygotowuje projekt do pracy na jednym repozytorium.
