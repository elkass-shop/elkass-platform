# Migracja i praca z GitHub / Vercel

Repo docelowe: `elkass-shop/elkass-platform`.

## Zasady pracy

1. Pracujemy na jednym repozytorium.
2. Nie tworzymy nowych repo dla każdej wersji.
3. Każdy sprint trafia jako commit.
4. Vercel automatycznie wdraża zmiany po `push`.

## Jak wgrywać poprawkę

1. Rozpakuj paczkę z poprawionym projektem.
2. Zastąp pliki w lokalnym repo.
3. Sprawdź zmiany w GitHub Desktop.
4. Commit, np. `Sprint V44 product polish`.
5. Push origin.
6. Sprawdź stronę na Vercel.
